<objective>
Close the one existing depth gap TSADAPT-01 targets on two already-⭐/🟢 languages at once: TypeScript
and JavaScript (via the shared `extract_ecmascript` core) currently have no entry-point detection at
all, unlike Rust/Python/Go/Java. Port the two already-proven in-repo patterns — Python's
`PY_ROUTE_VERBS` call-terminal match and Java's `JAVA_ROUTE_ANNOTATIONS` decorator-terminal match — to
TS's verified AST shapes for Express/Fastify/Koa/Hono verb calls and NestJS decorators, per D-09/D-10/
D-11. Resolve, explicitly, the one real architecture gap research found: TS emits no per-method
`Symbol` for class members, so a NestJS method-level `@Get()` decorator has nothing Java-style to
attach to — this plan adopts the lower-scope option (aggregate route markers onto the class's existing
`Symbol`), matching Phase 1's explicit boundary against new-language-extractor-scale changes.

Purpose: this is a zero-grammar-risk, high-confidence-builder win, and closes an honest gap on two
of the project's highest-trust (⭐/🟢) rows before the riskier extractor phases begin.

Output: entry-point detection implemented once in `extract_ecmascript` (both TS and JS inherit it),
unit-tested against both a named-handler and a decorator-based fixture, and the TypeScript
`Entry-pts` docs cell filled in.
</objective>

(call_expression
  function: (member_expression object: (identifier) property: (property_identifier))

  decorator: (decorator (call_expression function: (identifier) arguments: (arguments (string ...))))

    (method_definition name: (property_identifier) parameters: (...) body: (...)))))

// tree-sitter allows a field name to repeat across children in order.

    ```rust
    /// Terminal property/identifier names that indicate an HTTP-route call
    /// (Express/Fastify/Koa/Hono `<receiver>.<verb>(path, handler)` shape). Mirrors
    /// Python's `PY_ROUTE_VERBS`, plus `all` (Express/Fastify's "match any HTTP
    /// method" registration — genuinely route-shaped, unlike `.use()`).
    /// Deliberately EXCLUDES `use`: Express/Koa's generic middleware-registration
    /// call (logging, auth, static files — not HTTP routing); including it would
    /// flood `HttpRoute` markers with false positives on every Express/Koa app.
    const TS_ROUTE_VERBS: &[&str] = &[

        "route", "websocket", "ws", "all",
    ];
    ```

    Add a new function, called from `extract_ecmascript` right after `let mut symbols = defs;` (before
    `mod_sym` is pushed, so lookups only match real top-level definitions, not the module symbol
    itself):
    ```rust
    /// Walk the whole tree for Express/Fastify/Koa/Hono-shaped route-registration
    /// calls and, where the handler argument is a bare identifier matching an
    /// already-collected top-level `Symbol`'s name, attach an `EntryPoint::HttpRoute`
    /// marker to that symbol. Inline (arrow/function-expression) handlers are NOT
    /// detected — there is no per-call or per-inline-function `Symbol` in this
    /// extractor to attach the marker to, and fabricating one would violate the
    /// "never guess" contract. This is an honest, documented ceiling, not a gap.
    fn attach_call_entry_points(root: &Node, bytes: &[u8], symbols: &mut [Symbol]) {
        // walk `call_expression` nodes recursively; for each:
        //   - function: field must be `member_expression` (terminal = `property:` field text)
        //     or bare `identifier` (terminal = its own text) — same two-arm match as Python's
        //     `entry_points_for`'s `attribute`/`identifier` split.
        //   - terminal must be in TS_ROUTE_VERBS.
        //   - the LAST child of `arguments:` must be a bare `identifier` node — its text is the
        //     candidate handler name.
        //   - if a symbol in `symbols` has `.name == candidate` (name lookup, not node identity —
        //     there is no direct AST link from the call site to the definition), push
        //     `EntryPoint::HttpRoute(<full callee text, e.g. "router.get">)` onto that symbol's
        //     `entry_points`. If no match, or the last argument isn't a bare identifier, skip —
        //     do not create a marker with no attachment point.
    }
    ```
    Wire the call in `extract_ecmascript`:
    ```rust
    let mut symbols = defs;
    attach_call_entry_points(&root, bytes, &mut symbols);
    let mod_sym = super::module_symbol(lang, &namespaces, file, source.len());
    // ... (unchanged below)
    ```
