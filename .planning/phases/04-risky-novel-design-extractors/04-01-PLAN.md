**`.ml` symbol collection** (verified AST, 04-RESEARCH.md OCaml Code Examples section):
- `open_module module: (module_path (module_name))` / `include_module module: (module_path (module_name))` → walk recursively, emit `RefRole::Import` via `push_import_ref` (name = the `module_name`'s text).
- `module_definition (module_binding (module_name) body: (structure ...))` → `module_name` is a
  POSITIONAL first child (NOT a field — Pitfall 5). Emit `SymbolKind::Module` +
  `Descriptor::Namespace(name)`, then recurse into the `body: (structure ...)` children under an
  EXTENDED namespace (current namespaces + this module's name) so nested `let`/`type` get properly
  qualified descriptors.
- `module_type_definition (module_type_name) body: (signature ...)` → `module_type_name` also
  positional. Emit `SymbolKind::Interface` + `Descriptor::Type(name)` (closest existing kind to "a
  named signature", per research's Claude's-discretion recommendation).
- `type_definition (type_binding name: (type_constructor) body: ...)` → `name:` IS a field here. Emit
  `SymbolKind::TypeAlias` + `Descriptor::Type(name)`.
- `value_definition (let_binding pattern: (value_name) (parameter pattern: (value_pattern)) ... body: ...)`
  → if ANY direct child of the `let_binding` has kind `"parameter"` → `SymbolKind::Function` +
  `Descriptor::Method { name, disambiguator: String::new() }`; else → `SymbolKind::Static` +
  `Descriptor::Term(name)`. `pattern: (value_name)` IS a named field for the binding's own name.

Use the `objc.rs`-style `seen: HashSet<String>` + `push_symbol` dedup wrapper (first-occurrence-wins
by SCIP id) for every emitted symbol — cheap insurance, matches project convention even though OCaml
has no expected same-name collisions in table-stakes scope.

**`.mli` symbol collection** (verified: DIFFERENT node kinds from `.ml`, per research — do not try to
reuse the `.ml` collection function as-is):
- `value_specification (value_name) type: (function_type domain: (...) codomain: (...))` at the
  top level (NOT nested inside a `module_type`'s `signature`) → `SymbolKind::Function` if `type:`'s
  child kind is `"function_type"`, else `SymbolKind::Static`. Descriptor: `Method`/`Term` respectively.
- `module_definition (module_binding (module_name) module_type: (signature (value_specification ...)))`
  — the KEY divergence from `.ml`: `.mli` module bindings use a `module_type:` field (the `: sig ... end`
  form) instead of `.ml`'s `body:` field (the `= struct ... end` form). Emit the same `Module` symbol
  shape, then recurse into `module_type: (signature ...)`'s children for nested `value_specification`s
  under the extended namespace.
- `type_definition` — same shape as `.ml`, reuse directly.
