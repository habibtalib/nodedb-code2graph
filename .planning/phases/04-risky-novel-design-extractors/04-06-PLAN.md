**Namespace derivation**: `haskell_namespaces(file)` — prefer the module header's own dotted name
(`header module: (module (module_id))`, joining `module_id` children with `.`); fall back to
path-derived namespace (strip `.hs`, strip `src/`, split on `/`) when absent.

**Visibility environment**: if the module `header` has an `exports:` field, collect every exported
name's text (from `export: (export variable: (variable))` and `export: (export type: (name) ...)`
children) into a `HashSet<String>`; a top-level symbol's visibility = `Public` if its name is in that
set, else `Private`. If NO `exports:` field is present at all → every top-level symbol is `Public`
(D-08's explicit rule — do not default to Private).

**Symbol collection — two-pass grouping** (verified AST, 04-RESEARCH.md "Haskell" Code Examples +
Architecture Patterns "Haskell multi-equation bindings" sections):
- Walk the module's `declarations` (and, separately, each `instance`'s `instance_declarations`) for
  `function name: (variable) patterns: (...) match: (...)` and `bind name: (variable) match: (...)`
  nodes (the latter has NO `patterns:` field — a zero-argument binding; both kinds must be handled).
- For each such node, use the STANDARD `seen: HashSet<String>` + `push_symbol` dedup wrapper (exactly
  like `erlang.rs`'s clause-grouping, but with `disambiguator: String::new()` — Haskell needs no arity
  discriminator, there is no arity-based overload identity here) — sibling equations of the same name
  collapse to the FIRST occurrence's span/line automatically.
- `data_type name: (name) constructors: (...)` → `SymbolKind::Enum`.
- `newtype name: (name) constructor: (...)` → `SymbolKind::Struct`.
- `class name: (name) ...` → `SymbolKind::Trait`.
- `instance name: (name) patterns: (type_patterns (name)) declarations: (instance_declarations ...)` →
  emit a `RefRole::IsImplementation` reference to the class `name`'s text via `push_ref`, THEN recurse
  into `instance_declarations`'s `function`/`bind` children using the SAME two-pass grouping (separate
  `seen` scope from the top level — an instance method sharing a name with a top-level binding is a
  distinct symbol, nested under `Descriptor::Type` of... actually Haskell instance methods are NOT
  separately namespaced by instance in this v1 model; emit them as ordinary top-level-shaped Function
  symbols too, consistent with "the instance declaration is a static, verifiable fact; call-site
  dispatch is not" — document this simplification in the module doc).
- Imports: `imports: (imports import: (import module: (module (module_id) ...) names: (import_list ...))? )`
  → for each `import`, join the `module_id` children's text with `.` and emit ONE `RefRole::Import`
  reference for the module path (regardless of `qualified`/`hiding` presence — see ceiling above).

**Calls — mandatory recursive walk, NOT a flat Query** (per Anti-Patterns: nesting is unbounded, one
call site per curried argument):
```rust
/// Recurse into an `apply` node's `function:` field (Haskell DOES label this field, unlike F#)
/// until reaching a non-apply leaf; returns that leaf (the applied head).
fn application_head<'t>(node: &Node<'t>) -> Option<Node<'t>> {
    if node.kind() == "apply" {
        let inner = node.child_by_field_name("function")?;
        return application_head(&inner);
    }
    if matches!(node.kind(), "variable" | "constructor") {
        return Some(*node);
    }
    None
}
```
Walk the whole tree; for every `apply` node whose PARENT is NOT itself an `apply` (process only the
outermost node of each curry chain), call `application_head` and emit a Call reference for a
`variable` leaf (constructors are data construction, not calls — skip those). Recurse into the
`argument:` field (and past the head) so nested calls (`f (g x)`) are still found. Do NOT attempt to
model `infix`/`operator` nodes (`$`, `.`, user operators) as calls — explicitly out of scope (deferred
idea).
