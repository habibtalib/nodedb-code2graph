**Calls — mandatory recursive walk, NOT a flat Query** (per 04-RESEARCH.md Architecture Patterns
"F# and Haskell nest curried application" + Anti-Patterns "Writing a flat CALL_QUERY... for F#/Haskell
juxtaposition"):
```rust
/// Recurse into an `application_expression`'s first (function-position) child until
/// reaching a non-application leaf; returns that leaf node (the applied head).
/// F#'s grammar gives NO field labels here (unlike Haskell's `function:`/`argument:`),
/// so this walks POSITIONAL first children only.
fn application_head<'t>(node: &Node<'t>) -> Option<Node<'t>> {
    if node.kind() == "application_expression" {
        let first = node.child(0)?;
        return application_head(&first);
    }
    if matches!(node.kind(), "long_identifier_or_op" | "long_identifier") {
        return Some(*node);
    }
    None
}
```
Walk the whole tree recursively; for every `application_expression` node whose PARENT is NOT itself an
`application_expression` (process only the outermost node of each curried chain, to avoid emitting the
same call once per curry level), call `application_head` and, if it resolves to a
`long_identifier_or_op`/`long_identifier` with one or more `identifier` children, emit a Call reference:
all-but-last identifier segments joined as `qualifier` (`None` if only one segment), last segment as
`name`. Still recurse into the application's OTHER (argument) children so nested calls
(`f (g x)`) are not missed.
Read `infix_op`/`operator` child TEXT to detect `|>` only if trivially convenient; do not build special
pipe-desugaring logic beyond emitting the ordinary Call for whatever `apply`/`application_expression`
the pipe's right-hand side already contains (the generic walk already reaches it — no extra code
needed; document this simplification in the module doc alongside the ceiling notes).
