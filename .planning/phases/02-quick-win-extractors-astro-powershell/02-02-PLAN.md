**Frontmatter discovery** (verified AST, 02-RESEARCH.md "Pattern 5"): a `.astro` document has AT MOST
ONE `frontmatter` node (direct child of `document`), entirely absent when there's no fenced
frontmatter block (Pitfall 3 — treat as `Option`, never `.unwrap()`). When present, it contains a
single `frontmatter_js_block` raw-text child (analogous to `raw_text` under `script_element`):
```rust
fn find_frontmatter<'a>(root: &Node<'a>) -> Option<Node<'a>> {
    root.children(&mut root.walk()).find(|n| n.kind() == "frontmatter")
}
fn frontmatter_js_block<'a>(frontmatter: &Node<'a>) -> Option<Node<'a>> {
    frontmatter.children(&mut frontmatter.walk()).find(|n| n.kind() == "frontmatter_js_block")
}
```
**Critical:** frontmatter has NO `start_tag`/attributes at all — there is no `lang` to detect. ALWAYS
call `extract_ecmascript(inner_source, file, Language::TypeScript)` for the frontmatter block
(Astro compiles frontmatter as TS unconditionally). Do NOT attempt `detect_script_lang` on the
`frontmatter` node (anti-pattern flagged in research).

**`<script>` blocks:** port `collect_script_elements`, `find_raw_text`, and `detect_script_lang` from
`svelte.rs` VERBATIM (confirmed identical node-kind literals: `script_element`, `raw_text`,
`start_tag`, `attribute`, `attribute_name`, `quoted_attribute_value`, `attribute_value`) — zero
changes to any string literal.

**Merge loop:** copy `svelte.rs::extract`'s merge shape exactly: push a document-spanning root
`ScopeKind::Module` scope at index 0 (`doc_root`), then process the frontmatter block (if present)
and every `script_element` block through the SAME merge steps svelte.rs uses — `delta =
raw_text.start_byte()` (or `frontmatter_js_block.start_byte()` for frontmatter),
`extract_ecmascript` → `shift_offsets(&mut block_facts, delta, file, "astro", bytes)` → shift
`scope`/`parent` indices by `scope_base = merged.scopes.len()` → re-parent the block's former root
scope under `doc_root` → filter out `SymbolKind::Module` symbols from the block before extending
`merged.symbols`. After the loop, synthesize exactly one `module_symbol(Language::Astro,
&module_namespaces(file), file, source.len())` and push it. The frontmatter block is just one more
"block" in this loop — process it first (or last; order doesn't matter for correctness), the only
difference from a `script_element` block being its fixed TypeScript language and its
`frontmatter_js_block`-based text lookup instead of `raw_text`.

A no-frontmatter, no-script document (Pitfall 3's `<main><p>Hello</p></main>` case) must still reach
the "push doc_root scope, run zero merge iterations, synthesize one Module symbol" path cleanly —
this is `svelte.rs`'s `no_script_block_emits_single_module_symbol_and_root_scope` test shape, ported.

**Wiring:** in `src/extract/mod.rs`, add `#[cfg(feature = "astro")] pub mod astro;` and
`#[cfg(feature = "astro")] pub use astro::AstroExtractor;`. In `src/extract/dispatch.rs`, add
`#[cfg(feature = "astro")] use super::AstroExtractor;` and
`#[cfg(feature = "astro")] Language::Astro => AstroExtractor.extract(source, file),` to the match.
