  <action>
    Two independent one-line fixes, both required before Task 2 can wire any grammar-only candidate:

    1. In `src/grammar.rs`, remove the `#[cfg(feature = "_extractors")]` line directly above
       `use tree_sitter::Language;` so the import becomes unconditional:
       ```rust
       use tree_sitter::Language;
       ```
       `tree_sitter` is a mandatory (non-optional) crate dependency — there is no cost to always
       importing it. Without this fix, every grammar-only candidate feature (no `_extractors`, no
       `Language` enum variant) fails `cargo check --no-default-features --features <candidate>`
       with `error[E0425]: cannot find type 'Language' in this scope`.

    2. In `Cargo.toml`, fix the pre-existing `luau` feature declaration from:
       ```toml
       luau = ["dep:tree-sitter-luau", "_extractors"]
       ```
       to:
       ```toml
       luau = ["dep:tree-sitter-luau", "lua", "_extractors"]
       ```
       `src/extract/luau.rs` imports `super::lua::extract_lua_family` (structurally reuses the Lua
       extractor), but the `luau` feature never declared a dependency on the `lua` feature — an
       isolated `luau`-only build fails with `error[E0432]: unresolved import 'super::lua'`. This is
       not new-work regression; it is a pre-existing gap on `main` that COMPAT-03's CI job (01-03)
       will immediately catch unless fixed here first.
  </action>
  <acceptance_criteria>

    - `cargo test --workspace --all-features` exits 0 (no regression across the existing 22 languages)
