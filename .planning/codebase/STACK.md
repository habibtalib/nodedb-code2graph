## Configuration

**Rust Toolchain:**
- Edition: 2024
- MSRV: 1.85 (workspace-wide, enforced by CI)
- Default features: all 24 language grammars enabled (lines 24)

**Cargo Features:**

- `default`: All 24 language grammars + core extraction
- `manifest`: Enable TOML/JSON parsing for manifest inspection (optional, includes `serde`, `serde_json`)
- `serde`: Enable serde serialization (used by Python/Node FFI)
- `_extractors`: Internal feature; auto-enabled by every language feature
- Per-language: `rust`, `python`, `typescript`, `go`, `java`, `c`, `cpp`, `ruby`, `php`, `shell`, `swift`, `kotlin`, `solidity`, `sql`, `hcl`, `csharp`, `scala`, `dart`, `lua`, `luau`, `pascal`, `svelte`

**Build Profile (release):**
```toml
[profile.release]
opt-level = 3           # Maximum optimization
lto = "thin"           # Thin link-time optimization
codegen-units = 1      # Slower compile, faster binary
```

**Bindings Configuration:**

*Python (maturin):*
- Build backend: `maturin` (>=1.7,<2.0)
- Module name: `code2graph` (bindings/python/pyproject.toml)
- Output: PyPI wheel artifacts for py39+, plus source distribution (sdist)

*Node.js (napi-rs):*
- Binary name: `code2graph-node` (bindings/node/package.json napi.binaryName)
- Targets: x86_64/aarch64 Linux (gnu/musl), x86_64/aarch64 macOS, x86_64 Windows MSVC
- Build: `napi build --platform --release`
- Output: Platform-specific `.node` native modules

## Platform Requirements

**Development:**
- Rust 1.85+ (dtolnay/rust-toolchain used in CI)
- Python 3.9+ (for testing maturin wheels)
- Node.js 22 (for building and testing napi bindings)
- C11 compiler (tree-sitter grammars use C99 for-loops and C11 static_assert)
  - Linux cross-compile: manylinux docker sets `-std=gnu11` for C99 support
  - musl builds: cargo-zigbuild with zig compiler (no -std flag needed)
  - Windows: MSVC (no flag needed, MSVC defaults to C11+)

**Workspace Members:**
- `code2graph`: Core library (`src/`)
- `eval`: Evaluation harness / scorecard (`eval/Cargo.toml`)
  - Unpublished; demonstrates public API only (no internal coupling)
  - Optional feature `oracle-regen`: regenerate SCIP oracle from committed index
- `bindings/python`: PyO3 FFI wrapper (`bindings/python/`)
- `bindings/node`: napi-rs FFI wrapper (`bindings/node/`)

**Distribution Targets:**
- Crates.io: `code2graph` crate (Rust library)
- PyPI: `code2graph-rs` package (Python wheels for py39+, Linux/macOS/Windows)
- npm: `@nodedb-lab/code2graph` scoped package (Node.js native modules)
