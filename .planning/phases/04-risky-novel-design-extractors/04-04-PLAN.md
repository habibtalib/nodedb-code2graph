mandatorily.

  <action>
Create `src/extract/elixir.rs`. Module doc header: state the load-bearing structural fact FIRST —
`defmodule`/`def`/`defp`/`defmacro`/`alias`/`import`/`require`/`use` are ALL the same `call` node kind
(`target: (identifier) ...`); the ONLY distinguishing signal is the `target` identifier's literal TEXT.
Document D-05's CEILING: macro/`use` expansion never attempted (the call/use site is recorded, nothing
expanded); arity IS part of function identity (`Descriptor::Method { disambiguator: arity }`, chosen
per research recommendation — document this choice explicitly in the doc comment, matching the D-05
discretion requirement).

**Namespace derivation**: `elixir_namespaces(file)` — strip `.ex`/`.exs`, strip `src/`/`lib/` prefix,
split on `/` (fallback path; `defmodule`'s own dotted name is the PRIMARY namespace source per module,
same "prefer explicit declaration, fall back to path" pattern as `module_symbol`/`fortran_namespaces`).

**Manual recursive walk — NOT a flat Query** (the def/defp distinction cannot be expressed as a tree-
sitter node-kind pattern; see Anti-Patterns in 04-RESEARCH.md): write
`fn walk(node: &Node, ctx: &ExtractCtx, namespaces: &[String], enclosing_type: Option<&str>, seen: &mut HashSet<String>, out_syms: &mut Vec<Symbol>, out_refs: &mut Vec<Reference>)`
that recurses the whole tree. On a `call` node, read its `target` field's text:
- `"defmodule"` → the first `arguments` child's `alias` node's raw text (dotted, single leaf, no
  children to split — verified) becomes the module's dotted name; emit `SymbolKind::Module` +
  `Descriptor::Namespace` chain (split the dotted text on `.` into successive `Namespace` segments),
  then recurse into the `do_block`'s children under the EXTENDED namespace.
- `"def"` / `"defp"` → the FIRST `arguments` child is itself a `call` node — ITS OWN `target` is the
  function name, ITS OWN `arguments` children count = arity. Emit `SymbolKind::Function` with
  `Descriptor::Method { name: fn_name, disambiguator: arity.to_string() }`, visibility `Public` for
  `"def"` / `Private` for `"defp"`. Use the `objc.rs`-style `seen: HashSet<String>` dedup — THIS TIME
  IT IS LOAD-BEARING, not insurance: it is what makes multiple clauses of the same name+arity collapse
  to one symbol while different-arity overloads (different SCIP id) both survive. Recurse into the
  `do_block` for nested calls (still under the enclosing module's namespace, no additional nesting).
- `"defmacro"` → same shape as `def`, but emit `SymbolKind::Function` + BOTH
  `Descriptor::Method { name, disambiguator: arity.to_string() }` AND note in a comment that macro call
  SITES elsewhere are recorded as ordinary Calls (no special macro-call node exists in Elixir the way
  Julia's `macrocall_expression` does) — do not fabricate one.
- `"alias"` / `"import"` / `"require"` / `"use"` → the first `arguments` child (an `alias` node, or a
  nested `call`/`dot` for `Module.Sub` forms) — read its raw text as the imported module path, emit
  `RefRole::Import` via `push_import_ref`. Table stakes: capture the primary module path only; `as:`/
  `only:`/`except:` keyword-list options are NOT modeled (ceiling — document).
- Any OTHER `call` node (not matching the above target texts): if `target` field's kind is `"dot"`
  (`left: (alias) right: (identifier)`, e.g. `Module.fun(args)`) → emit a Call reference with
  `qualifier` = the `left` alias's text, `name` = the `right` identifier's text. Else if `target` kind
  is `"identifier"` → emit a plain Call reference (local function call), `name` = target text.

Pipe (`|>`) handling: a `binary_operator` node's operator token TEXT (read as an unnamed child between
`left`/`right`) — if it equals `"|>"`, no special call-emission is needed beyond normal recursion into
`right` (which, if it's itself a `call`, is already visited and emits its own Call reference via the
generic dispatch above). Document this simplification explicitly in the module doc rather than
building bespoke pipe-desugaring logic.
