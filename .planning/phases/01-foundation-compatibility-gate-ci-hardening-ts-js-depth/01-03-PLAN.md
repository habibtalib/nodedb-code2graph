<objective>
Close the pre-existing CI gap COMPAT-03 targets: no job today builds any single language feature in
isolation (`grep -r no-default-features .github/workflows/` returns nothing) — the only defense against
a feature-flag combinatorics break is `--all-features`, which can mask exactly the kind of isolation bug
01-01 found in `luau` and the `_extractors` import gate. Add one ubuntu-only matrix job per D-07/D-08
that runs `cargo check --no-default-features --features <lang>` for every language feature that exists
after 01-01/01-02 land (22 pre-existing + the candidates that passed), so a future isolation break fails
the pipeline instead of silently compiling only under `--all-features`.

Purpose: this is the CI-hardening half of Phase 1's mandate — every later phase (2-4) adds more
language features, and without this job each one recreates the same undetected-isolation-break risk
01-01 discovered by hand.

Output: a new `feature-isolation` job in `.github/workflows/test.yml`, verified locally (not just
trusted to CI) against the full current feature list.
</objective>

  <action>
    Add a fourth job to `.github/workflows/test.yml`, after the existing `bindings` job, per D-07/D-08:
    ubuntu-only, `fail-fast: false` matrix (so one language's break doesn't cancel the others — full
    per-language failure attribution in the PR UI), using `cargo check` (not `cargo test`) to keep it
    cheap since this job only proves standalone compilation, not correctness (correctness is already
    covered by the `test` job's `--all-features` run and each candidate's own ABI test in 01-01/01-02).

    First, derive the current, complete language-feature list from `Cargo.toml` (do not hand-type it
    from the plan-time candidate list — 01-02's disputed-4 outcomes may have changed the final count):
    ```bash
    awk '/^\[features\]/{f=1;next}/^\[/{f=0}f' Cargo.toml \
      | grep -oE '^[a-z_]+' \
      | grep -vE '^(default|manifest|serde|_extractors)$'
    ```
    This yields the 22 pre-existing languages plus whichever of the 11+4 candidates from 01-01/01-02
    are actually wired (failed/reverted candidates are absent from `[features]` entirely, so they
    correctly drop out of this list with no manual bookkeeping).

    Add the job using that derived list as `matrix.lang`:
    ```yaml
      feature-isolation:
        name: Feature isolation (${{ matrix.lang }})
        runs-on: ubuntu-latest
        strategy:
          fail-fast: false
          matrix:
            lang: [rust, python, typescript, go, java, c, cpp, ruby, php, shell, swift,
                   kotlin, solidity, sql, hcl, csharp, scala, dart, lua, luau, pascal, svelte,
                   <candidates actually present after 01-01/01-02, derived above>]
        steps:
          - uses: actions/checkout@v6
          - name: Install Rust
            uses: dtolnay/rust-toolchain@stable
          - uses: Swatinem/rust-cache@v2
            with:
              prefix-key: isolation-${{ matrix.lang }}
          - name: Build ${{ matrix.lang }} in isolation
            run: cargo check --no-default-features --features ${{ matrix.lang }}
    ```
    The `prefix-key: isolation-${{ matrix.lang }}` is required (not optional) — without a per-matrix-
    value cache key, `Swatinem/rust-cache` thrashes across differently-featured builds sharing one
    cache, per 01-RESEARCH.md Architecture Patterns §3.

    Do not modify the `lint`, `test`, or `bindings` jobs — this is strictly an additive fourth job.
